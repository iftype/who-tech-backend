who.tech 사용 가이드

우아한테크코스 크루 단체 서비스 이용 방법

1 크루 검색하기

좌측의 검색창에 크루명 또는 GitHub ID를 입력하여 크루를 검색할 수 있습니다. 입력 후 300ms의 딜레이 후 자동으로 검색 결과가 표시됩니다.

  • 기수 버튼, 역할(크루/코치/리뷰어), 트랙 정보 드롭다운을 함께 활용할 수 있습니다
  • 크루를 클릭하여 상세 페이지로 이동합니다
  • 3FK 단축키로 빠르게 검색창에 포커스할 수 있습니다

💡 팁

여러 필터를 조합하여 특정 기수, 역할, 트랙의 크루들을 효율적으로 찾을 수 있습니다.

2 상세 페이지 — 미션 아카이브 & 블로그

크루의 전체 미션 PR 아카이브를 기수와 레벨별로 확인할 수 있습니다. 각 미션의 PR 링크를 통해 직접 코드 리뷰를 참고할 수 있습니다.

📋 아카이브 보기

  • 기수별로 정렬된 미션 목록을 확인할 수 있습니다
  • 각 기수의 정해진 레포만 표시됩니다 (CohortRepo 정의 필수)
  • 프리코스(Precourse)와 정규 미션을 구분하여 표시합니다

📝 블로그

  • 크루가 설정한 블로그의 최근 글 10개를 표시합니다
  • RSS 피드를 통해 자동으로 수집됩니다
  • GitHub 프로필의 bio에서 블로그 링크를 감지합니다

📌 마크다운 목차

'마크다운 목차' 버튼으로 아카이브를 마크다운 형식의 포트폴리오 목차로 내보낼 수 있습니다. README나 포트폴리오에 바로 복사할 수 있습니다.

3 GitHub에 블로그 링크 설정하기

who.tech에서 블로그 글을 표시하려면 GitHub 프로필에 블로그 링크를 등록해야 합니다.

📚 설정 방법

1단계: GitHub 프로필 편집

GitHub 프로필 설정에서 "Bio" 필드를 엽니다.

2단계: 블로그 URL 추가

Bio 필드에 블로그 URL을 포함시킵니다.

예시: 안녕하세요! 우아한테크코스 크루입니다. 블로그: https://myblog.com

3단계: 저장

변경 사항을 저장합니다. who.tech은 24시간 이내에 자동으로 블로그 정보를 감지하고 동기화합니다.

⚠️ 주의: 블로그 URL은 https:// 또는 http://로 시작해야 합니다. 또한 유효한 RSS 피드가 있어야 합니다.

🔗 지원되는 형식

  • 개인 블로그 (RSS 지원) - Tistory, Velog, Medium 등
  • GitHub Pages (Jekyll RSS)
  • Notion (공개 페이지 + RSS)

4 RSS 피드 안내

who.tech은 RSS 피드를 통해 블로그 글을 자동으로 수집합니다. 블로그에 새 글이 발행되면 최대 1시간 이내에 who.tech에 반영됩니다.

📡 RSS 피드란?

  • RSS는 블로그의 업데이트 정보를 자동으로 수집하는 표준 형식입니다
  • 대부분의 블로그 플랫폼(Tistory, Velog, Medium 등)이 자동으로 지원합니다
  • who.tech이 자동으로 감지하면 설정 불필요합니다

🔍 RSS 피드 찾기

Tistory 블로그

https://[블로그명].tistory.com/rss

Velog

https://velog.io/rss/@[사용자명]

Medium

https://medium.com/feed/@[사용자명]

GitHub Pages (Jekyll)

https://[username].github.io/feed.xml

💡 팁: 블로그에 RSS 피드가 없으면 who.tech에서 자동으로 수집할 수 없습니다. 블로그 플랫폼의 설정에서 공개 RSS를 활성화하세요.

5 기수별 목록

좌측 메뉴에서 기수를 선택하면 해당 기수의 모든 크루를 한눈에 볼 수 있습니다.

  • 기수 내 크루의 미션 완료율을 확인할 수 있습니다
  • 각 크루의 역할(크루/코치/리뷰어)이 표시됩니다
  • 크루 카드를 클릭하여 상세 페이지로 이동합니다

🎯 활용 팁

기수별 목록에서 모르는 크루와 친해질 기회를 만들어보세요!

6 피드 — 최신 블로그 글

모든 크루의 최신 블로그 글을 한 곳에서 볼 수 있습니다.

  • 실시간으로 업데이트되는 크루들의 블로그 글 피드
  • 기수와 트랙별로 피드를 필터링할 수 있습니다
  • 각 글의 공개 여부는 크루의 GitHub 프로필 설정에 따릅니다

📌 팁

피드에서 다른 크루의 학습 경험과 인사이트를 공유받을 수 있습니다.

질문이나 버그 제보는 GitHub Issues에 등록해주세요.
